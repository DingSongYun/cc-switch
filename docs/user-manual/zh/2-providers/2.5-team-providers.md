# 团队供应商配置

团队供应商功能允许管理员维护一份远程 **Team Provider Registry** JSON，成员在 CC Switch 中配置 Registry URL 后拉取并合并到本地供应商列表。

## 适用场景

- 团队统一维护 NewAPI / 网关的 endpoint、模型、协议格式
- 成员各自填写个人 API Key，避免密钥进入共享配置
- 比 WebDAV/S3 整库同步更细粒度，不会覆盖当前选中供应商、代理开关等设备级设置

## 管理员：发布 Registry

1. 参考示例文件 [`docs/examples/team-provider-registry.json`](../../examples/team-provider-registry.json)
2. 将 JSON 发布到 HTTPS 可访问地址（Git raw、对象存储、内网静态站点均可）
3. 不要在 Registry 中写入明文 API Key（客户端会拒绝）

## 成员：启用同步

1. 打开 **设置 → 高级 → 团队供应商配置**
2. 填写 Registry URL
3. 可选开启自动同步间隔（分钟，0 为关闭）
4. 点击 **立即同步**
5. 在 Claude Code / Claude Desktop / Codex / Gemini 供应商列表中找到带 **团队托管** 标识的 provider，填写个人 API Key

## 冲突处理

若本地修改了团队托管字段（如 Base URL），再次同步时会出现冲突：

- **接受团队版本**：覆盖为 Registry 最新内容
- **保留本地副本**：跳过自动覆盖，字段保持本地值

## 清理

Registry 移除某供应商后，本地对应 `team-*` provider 会被标记为已移除。可在设置页点击 **清理已移除的团队供应商**（不会删除当前选中的 provider）。

## 手动测试清单

- [ ] 首次同步：Registry 生成 `team-claude-*` / `team-claude-desktop-*` / `team-codex-*` / `team-gemini-*`
- [ ] 填写 API Key 后重同步：密钥保留不变
- [ ] 修改 Registry baseUrl 后重同步：endpoint 更新、密钥仍保留
- [ ] 本地修改 baseUrl 后重同步：出现冲突并可接受/保留
- [ ] Registry 移除 app 或 provider id：本地标记 removed，cleanup 可清理
- [ ] WebDAV/S3 整库同步与 Universal Provider 仍正常工作
